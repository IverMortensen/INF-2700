db2700
A Tiny DBMS for Assignments in INF-2700

This project is used in the programming assignments of the Database course INF-2700.

The DBMS consists of three layers: pager, schema and frontend.

A database table, consisting of a sequence of data records (of fixed length), is stored in a file. Physically on disk, a file consists of a number of consecutive disk blocks and the data records are stored in these blocks. The structure of records are defined in the schema of the table.

To process data in a table, the data blocks are read into buffer pages in the maim memory. A pager provides the functions to work with the pages.

The schema layer, in addition to maintaining the schemas of tables, also handles table operations including display, search and project of a table.

A database user can run SQL-like commands through a frontend.

Programming physical data management involves tremendous efforts in memory management and debugging. For most of the data structures, you can find useful printing operations at your disposal.

The pager profiler helps you investigate the performance of various operations.

To learn how to use the pager, schema and record, read the source code in testschema.c and interpreter.c.
